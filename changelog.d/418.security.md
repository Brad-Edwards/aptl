### Security

Restored SSH host-key verification on the operator-facing WebSocket
terminal relay (ADR-040). The relay previously dialed `asyncssh` with
`known_hosts=None`, disabling server-identity verification on the
connection that carries the operator's interactive PTY input. `aptl lab
start` now pins each lab SSH endpoint's host key into an ignored
`.aptl/known_hosts` file (trust-on-first-use only during provisioning,
on the trusted host), and the relay verifies every connection against
that file — failing closed on a missing pin or a host-key mismatch.

The relay now derives each target's host/user/port from the canonical
endpoint registry projected over runtime inventory (container IP over the
bridge) instead of a hardcoded `localhost:<port>` map. This also repairs
the terminal for `victim`, `kali`, and `workstation`, whose published
host ports were removed in #293 — their `localhost` dials had been
failing to connect. `workstation` is now a registered SSH endpoint.
