**Per-function complexity gate now covers the TypeScript surfaces, with zero
carve-outs.** A hard, repo-wide ESLint `complexity` gate (cyclomatic, threshold
15 — the sibling of the Python `ruff` `C901` gate) runs in the required
*Pre-commit hooks* CI job and locally, failing any new function over the
threshold under `mcp/*/src/` or `web/src/`, including functions in Svelte
`<script lang="ts">` blocks; the threshold and scope live in the root
`eslint.config.js`. The eleven functions that were over threshold (eight Python,
up to CC 45, plus three TypeScript) were split into helper functions in the same
change, so neither gate ships with any per-file exemption — every function in
`src/`, `mcp/*/src/`, and `web/src/` is now gated at 15. See ADR-010.
