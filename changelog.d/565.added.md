### Added

- Web scenario workbench (`/scenarios/[id]`) now renders a notebook-style,
  document-first investigation surface from a new backend-owned ACES/catalog
  projection. A `GET /api/scenarios/{id}` endpoint returns scenario header facts
  plus an ordered `WorkbenchBlock` union (narrative, container status,
  objective, step, SIEM query, and lazy terminal blocks) projected from the
  curated catalog and the ACES SDL parser — not from the removed in-tree
  scenario schema. The catalog gains a narrow validated metadata extension
  (mode, difficulty, estimated duration, tags), and Lab Home scenario cards now
  show mode, difficulty, duration, required containers, tags, and validation
  state. Embedded terminal blocks stay lazy (no PTY until an explicit action)
  and SIEM query blocks stay display-only (execution is a later slice). (#565)
