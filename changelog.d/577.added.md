Declared ACES `account-placement` resources are now realized onto their target node by the deployment backend instead of only being recorded: `DeploymentBackend.realize_accounts` ensures the declared groups, creates or reconciles each user, applies the supported non-secret attributes (group membership, mail, SPN, disabled), and verifies the result by read-after-write against the resolved provider (Samba AD on the `ad` node). Realization is a convergent upsert — an already-existing account is reconciled, not re-created, so a provisioner-owned credential is preserved — and no credential ever crosses argv, environment, or run evidence: user creation delegates password generation to the target provider (ADR-029, ADR-046 Account and Identity Realization Addendum). The backend manifest now advertises only the account features it materializes and verifies.
