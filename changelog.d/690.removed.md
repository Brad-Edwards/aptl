The captured TechVault asset inventory (`docs/aces/inventory/`) and the SCN-010
parity-inventory surface (`docs/aces/parity-inventory.yaml`/`.md`, the
`check_parity_manifest` static-gate check, and the `required_surface_coverage`
contract) are removed from APTL, along with the `aptl aces-inventory` command
and the mapping-ledger validation behind it. That capture was an experiment run
inside APTL to prove out an asset-inventory capability; the capability now lives
in ACES, and APTL keeps only `scenarios/techvault-operational.sdl.yaml` as the
ACES-conformant driving contract, with no separate capture/parity evidence
surface behind it.

The static validation gate now runs parse, import lock, compile, backend
conformance, provisioning realization, and account provisioner parity, with
`GateOptions.phase` and the surface-coverage deferral contract removed. The
import-lock check now passes for a scenario that declares no `imports:` (there
is nothing to resolve, so there is no lockfile to verify) and stays fail-closed
for every scenario that does import.
