**Detected stale Wazuh Indexer credentials during `aptl lab start`.** When the indexer's listener is up but rejects the configured `.env` credentials (HTTP 401) — typically because a reused `wazuh-indexer-data` volume still holds the previous admin password — startup now emits a targeted diagnostic pointing at `aptl lab stop -v` recovery instead of a generic "did not become ready" timeout message.
