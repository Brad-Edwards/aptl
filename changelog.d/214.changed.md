**Lab orchestration steps now enforce their `_LabStartContext` preconditions with `icontract.require` instead of `assert`.** `start_lab`, `_step_sync_credentials`, `_step_pull_images`, `_step_start_containers`, `_step_wait_for_services`, `_step_test_ssh`, and `_step_seed_soc` now refuse to run when env / config / backend / SSH key has not been populated, regardless of whether Python is invoked with `-O`. The prime SOC seed additionally guards the parameterised required-profiles predicate (`wazuh, enterprise, victim, kali, fileshare, soc`). A contract breach is converted to a fatal `LabResult` with a narrow redacted error at the orchestrator boundary; raw violation text never reaches the CLI. Adds `icontract` as a runtime dependency.
