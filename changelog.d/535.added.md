### Added

- Recorded live-boot proof for the curated TechVault ACES startup variants
  (`techvault-observability-core`, `techvault-defensive-min`,
  `techvault-enterprise-web`, `techvault-attacker-target`). Each variant is
  booted through the public `aptl lab start --scenario <id>` path and its running
  containers and networks are checked against the variant's ACES-realized
  *reduced* Compose surface — not the full TechVault range. A new
  `aptl.validation.curated_live_proof` helper derives the expected reduced matrix
  from the same ACES realization the start path uses and compares it to the
  captured range snapshot, and the recorded evidence, exact reproduction
  commands, and known limitations live in
  [the curated live validation gate doc](docs/aces/techvault-curated-live-validation-gate.md).
  The full TechVault live validation gate (PR #520) is unchanged and remains the
  separate full-surface proof.
