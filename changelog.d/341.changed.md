Adopted the ACES `runtime.environment` secret-name rule across the TechVault
SDL: runtime environment variables whose names look secret-bearing (for
example `PWD`, `GPG_KEY`, `LABADMIN_SSH_KEY_FILE`) now omit their value and are
classified `redacted`, in the `wazuh.indexer` node generator and in the eight
hand-authored nodes that carried such values (`dns`, `mailserver`, `misp`,
`misp-db`, `victim`, `wazuh-manager`, `webapp`, `workstation`). Intentional
participant-visible lab credentials keep the `secret_fixture` classification.
A new `tests/test_techvault_classification_audit.py` guard fails if any node's
`runtime.environment` re-introduces a plain secret-named value, so the TechVault
SDL stays valid against the ACES rule.
