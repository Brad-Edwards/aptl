# Wazuh Indexer ACES Inventory Preflight

This note is the architecture preflight for SCN-010 / issue #341. It is a
binding guardrail for the `wazuh.indexer` steady-state inventory, not an
implementation plan and not a replacement for ADR-035, ADR-028, ADR-029, or
the ACES asset-inventory methodology.

## Architecture Decisions

- The completion artifact belongs under `docs/aces/inventory/wazuh.indexer/`
  and must use the existing inventory bundle shape: `README.md`,
  `capture-evidence.sh`, `mapping-ledger.yaml`, `evidence/`,
  `capture-limits.txt`, `captured-at-utc.txt`, and
  `evidence-sha256sums.txt`.
- The inventory describes the realized `aptl-wazuh-indexer` container at one
  post-`aptl lab start` steady-state snapshot. If the capture is not from a
  destructive fresh reset, record that boundary explicitly and do not claim
  clean-lab rebuild proof or byte-identical rebuildability.
- `scenarios/techvault.sdl.yaml` remains an ACES SDL document whose authority
  is the ACES parser, semantic validators, and runtime compiler. Do not
  validate or shape indexer additions through `aptl.core.sdl`,
  `aptl.core.scenarios`, or a new APTL mirror of ACES models.
- Keep these concepts separate: ACES-authored scenario facts, APTL Docker
  Compose realization, generated Wazuh PKI/config material, captured evidence,
  and future APTL backend consumption. Evidence and checksums prove facts; they
  do not substitute for SDL expression.
- `config/wazuh_indexer/` is source-owned configuration. The generated
  `config/wazuh_indexer_ssl_certs/` tree is lab-start material and is gitignored
  because it contains private key material. Do not infer certificate facts from
  a missing local generated directory.
- Missing ACES expressivity must become an upstream ACES blocker with checked
  current surfaces and evidence. APTL runtime-consumption gaps do not justify
  leaving observable indexer facts as evidence-only.

## Security And Validation Layers

- **ACES SDL shape:** indexer SDL additions must parse with the installed ACES
  parser, pass semantic validation, and compile through the ACES runtime
  compiler. The active ACES contract version is the gate; if a desired runtime
  family is absent or rejects the fact, file/link an ACES expressivity issue
  instead of adding `x-aptl-*`, comments, or a local schema.
- **Inventory ledger:** every captured fact needs an `AcesSurface` disposition,
  evidence reference, rationale, and either encoded fields, an explicit caveat,
  or a linked ACES/APTL gap. No `needs_gap_triage` row should remain at review.
- **Environment binding:** `INDEXER_USERNAME` and `INDEXER_PASSWORD` are owned
  by `.env` parsing through `load_dotenv` / `EnvVars`; placeholder rejection
  remains in `find_placeholder_env_values`. Do not add indexer-specific env
  parsing or move these values into SDL, parity inventory, or tests.
- **Secret classification:** ADR-029 is canonical. Raw `.env` values, bcrypt
  hashes when treated as credential material, private TLS keys, admin key files,
  session IDs, bearer tokens, cookies, and generated config secrets must not be
  committed unredacted. Encode path/metadata/checksum/classification where
  useful, with raw value omitted for `redacted` or `operator_secret` facts.
- **Generated PKI and bind mounts:** Wazuh certificates are generated by
  `ensure_ssl_certs` via `generate-indexer-certs.yml`, then consumed through
  Compose bind mounts. Capture may cite generated public certificates and
  private-key path/metadata, but must not commit private key bytes or invent
  cert material when `config/wazuh_indexer_ssl_certs/` is absent.
- **Config shape and path containment:** checked-in indexer files under
  `config/wazuh_indexer/` are source-owned; generated artifacts stay under
  existing generated-output ownership. Do not mutate checked-in config during
  capture. Do not introduce caller-provided paths that bypass containment,
  no-symlink, or bind-mount preflight expectations.
- **OS/process exposure:** capture and validation commands must not put real
  indexer passwords, private keys, API tokens, or auth headers in process argv,
  Docker healthcheck replacements, logs, exception text, or issue comments. Use
  redacted environment snapshots and safe curl patterns where credentials are
  needed.
- **Runtime/service exposure:** keep transport listeners, host-published port
  `9200`, OpenSearch transport ranges, Docker health, network attachment,
  restart/resource/ulimit policy, mounted volumes, container runtime policy,
  filesystem inventory, local identity, packages, SBOM components, scanner CVEs,
  and OpenSearch/Wazuh security logical state as distinct facts.
- **Error envelopes and observability:** any new test helper or CLI wrapper
  must report narrow diagnostics and reuse the shared redaction boundary. Raw
  Docker inspect payloads, scanner errors, ACES validation exceptions, and curl
  failures must not cross CLI, API, log, snapshot, or runstore boundaries
  unfiltered.

## Extensibility Seam

The seam is the per-asset inventory ledger plus ACES-native runtime fields for
one asset id and Compose service. A future Wazuh core asset, OpenSearch-backed
service, or profile upgrade should reuse the same bundle schema, evidence
script shape, redaction contract, and parser/compiler gates by changing asset
parameters, not by adding indexer-specific validator branches.

For SDL expression, parameterization belongs in ACES-native node source,
services, runtime container/network/mount/filesystem/software/package/security
state, relationships, and variables/imports where ACES supports them. Backend
realization remains behind the ACES backend profile and existing APTL
`DeploymentBackend` owners.

## Gotchas And Anti-Patterns

- Do not model the indexer as the Wazuh manager. Manager-side agents, rules,
  decoders, and API state stay under `runtime.security_monitoring_managers` on
  `wazuh-manager`; indexer-side OpenSearch/security/index state needs its own
  checked ACES surface or an ACES blocker.
- Do not flatten service listener `9200`, OpenSearch transport port ranges,
  Docker host publication, healthcheck command, and participant reachability
  into one generic "port" fact.
- Do not treat `wazuh/wazuh-indexer:4.12.0` as immutable. Pair the tag with
  observed digest/layers/history, package manifests, scanner versions, and
  capture time.
- Do not silently omit volume-backed data, generated security index state,
  logs, caches, shards, cluster UUIDs, or runtime-created files because values
  are unstable. Encode stable shape or record a first-class limit/blocker.
- Do not confuse checked-in `internal_users.yml` bcrypt hashes, `.env`
  passwords, OpenSearch internal roles, OS-local users, and ACES top-level
  accounts. They are different identity/authentication surfaces.
- Do not hide missing ACES expressivity in `metadata`, `properties`, comments,
  evidence-only ledger rows, or TechVault-name dispatch.
- Do not create a second scenario schema, second inventory validator, second
  secret taxonomy, second readiness taxonomy, or new exception hierarchy.
- Do not repair inventory findings by changing Docker Compose, Wazuh config,
  certificate generation, startup ordering, or endpoint behavior in this issue.

## Non-Goals

- Do not implement the inventory, capture evidence, edit TechVault SDL, file
  gap issues, or add tests from this preflight.
- Do not run `aptl lab stop -v && aptl lab start` unless the user explicitly
  authorizes destroying the current lab state.
- Do not implement the ACES backend interpreter, conformance runner, live lab
  deployment gate, default-scenario flip, scenario archive move, or deletion of
  `aptl.core.sdl` / `aptl.core.scenarios`.
- Do not redesign Wazuh TLS, generated cert ownership, `.env` handling,
  readiness classification, endpoint snapshots, run archives, or APTL backend
  profile conformance while inventorying the indexer.
