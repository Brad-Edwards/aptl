### Changed

**Made the target container images and their ACES inventories reproducible.**
The `victim`, `workstation`, and `reverse` Dockerfiles now pin the Falco
package version (`falco-0.44.1`) and set explicit file modes (`chmod 755` for
scripts, `COPY --chmod=644` for configs and unit files) instead of relying on
the build host's umask. Both changes stop the captured image — and therefore
the committed inventory — from churning between rebuilds and build
environments. A new `docs/aces/inventory/techvault-inventory-volatility.md`
records which inventory facts are pinned and which are captured by shape
because they churn per-build, per-recreate, per-boot, or per-database (Trivy);
the target inventory tests assert the volatile facts by shape rather than
value.
