**Added a static ACES validation gate for the TechVault scenario (SCN-010E).**
A new scenario-generic gate (`aptl.validation.techvault_gate`) composes the ACES
reference parser, import-lock verification, runtime compiler/semantic validator,
and canonical `backend-manifest-v2` conformance with APTL's provisioning
realization and the parity inventory. It proves `scenarios/techvault.sdl.yaml`
is parseable, semantically valid, and conformance-aligned, and that every
required observable surface is either represented with real compiled evidence or
explicitly deferred with a linked tracking issue. APTL's backend manifest is now
the canonical ACES `backend-manifest-v2` surface (`aces_backend_protocols`)
rather than an APTL-local shim. Fast gate-logic tests run in the default suite;
the full-scenario gate runs as an advisory CI job in Phase A and flips to
blocking at the SCN-010 cutover.
