Fixed `aptl lab start --scenario <id>` falsely reporting a bounded curated scenario as `degraded_unusable`. The post-start readiness checks (Wazuh service waits and SSH target probes) now scope to the profiles the scenario actually started rather than the global `containers` config flags, so a scenario that omits Wazuh, Kali, or the victim no longer waits on (and warns about) services it never launched.
