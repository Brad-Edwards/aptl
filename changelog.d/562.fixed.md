**APTL's ACES participant runtime now implements `admit_action`, restoring backend conformance.**
ACES (`aces@dev`) extended the participant-runtime backend protocol with an
`admit_action` method (participant implementation binding), which `AptlParticipantRuntime`
did not yet implement — so `create_aptl_runtime_target` failed the registry
target-contract check (`participant_runtime is missing callable method 'admit_action'`)
and every ACES backend, curated-variant, and TechVault gate test errored. The
runtime now admits an implementation-bound participant action by recording the
portable behavior-history events (action attempted, state-transition recorded,
observation emitted) against the live episode, and fails closed when there is no
live, non-terminal episode.
