### Added

- ACES SDL integration via explicit source-tree install (#310 /
  SCN-010 / ADR-035). aces-sdl is NOT yet declared in pyproject.toml;
  install via `git clone https://github.com/Brad-Edwards/aces.git
  ../aces-sdl && (cd ../aces-sdl && git checkout
  12d9a8234b354f4f181a5db251d791ac57441f52) && pip install -e
  ../aces-sdl/implementations/python`. The non-standard install
  recipe is forced by aces-sdl's source-tree-relative resource
  lookups; see `docs/lessons/2026-05-19-aces-backend-skeleton.md`.
  Phase B cutover formalizes the dependency once ACES ships a
  packaging layout that survives wheel install.
- `aptl.backends.aces` adapter — `AptlProvisioner`,
  `create_aptl_manifest`, `create_aptl_target`, `register` — declaring
  the `provisioning-only` ACES backend profile. `apply()` is a
  skeleton: empty plans succeed, non-empty plans raise
  `ApplyNotImplementedError` so the unwiring is explicit. Real
  reconciliation through `aptl.core.deployment.DeploymentBackend`
  lands in Phase A.2.
- Advisory CI job `aces-conformance` runs
  `aces_conformance.run_target_conformance` against the APTL target
  on every push, `continue-on-error: true` per #310 Phase A. Becomes
  blocking at Phase B cutover.
